# Implementation Plan: Go HTML Template Extraction with Live Reload

**Branch**: `015-extract-html-templates` | **Date**: 2026-07-09 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/015-extract-html-templates/spec.md`

## Summary

Move the three inline HTML string constants in `app/main.go` (`gettingStartedPageHTML`, `hostPageHTML`, `leaderboardPageTemplate`) to external `.html` files in a new top-level `templates/` directory, parsed with Go's standard `html/template` package (already used for `/play`). Extend the existing `/api/ping` live-reload mechanism to incorporate a per-process template version counter that is atomically incremented when any template file changes on disk (detected by a 1-second polling goroutine using `os.Stat`), enabling the browser to reload automatically within ~3–5 seconds of a template edit — with zero new Go dependencies.

## Technical Context

**Language/Version**: Go 1.25.0 (see `app/go.mod`)

**Primary Dependencies**: `html/template` (stdlib, already in use), `sync/atomic` (stdlib), `os.Stat` (stdlib)

**Storage**: Redis (unchanged) — no new storage requirements

**Testing**: Go standard testing + Testcontainers-go (unchanged pattern); template handler tests use `httptest.NewRecorder` with a real temp dir of template files

**Target Platform**: Linux container (Docker Compose, same as current stack)

**Project Type**: Web service (Go backend, Docker Compose orchestration)

**Performance Goals**: No change — template parse on each request is microseconds at demo-booth request volumes

**Constraints**: Zero new runtime dependencies (`go.mod` unchanged); must work with Docker Desktop volume mounts on macOS and Linux (rules out `fsnotify`)

**Scale/Scope**: Three pages converted; one already-working page (`/play`) unchanged; one new background goroutine; one new config env var

## Constitution Check

Done is not done until the live reload cycle has been observed in a browser: edit a template file → watch the browser reload without any `docker compose restart`. Quickstart Scenario 2 in `quickstart.md` defines the exact observation path.

No third-party assets added. All template files are project-authored HTML.

## Implementation Phases

### Phase A — Template Files

1. Create `templates/` directory at repo root.
2. Move `gettingStartedPageHTML` content → `templates/getting-started.html` (no dynamic fields; remove the `html/template` escaping if none used, but keep Go template syntax for consistency and forward-compatibility).
3. Move `hostPageHTML` content → `templates/host.html` (no dynamic fields).
4. Move `leaderboardPageTemplate` content → `templates/leaderboard.html` (dynamic: `.ScoresServiceURL`, `.CommitsServiceURL`).
5. Verify each file has valid Go `html/template` syntax.

### Phase B — App Changes

1. **Config**: Add `TemplatesDir string` to `Config`; add `TEMPLATES_DIR` env var to `loadConfig()` with default `/templates`.
2. **App struct**: Add `templatesDir string` and `templateVersion atomic.Int64` fields to `App`.
3. **Watcher goroutine**: Add `func (a *App) watchTemplates(ctx context.Context)` — polls mtime of each file in `a.templatesDir` every 1 second; on change, increments `a.templateVersion`.
4. **Start watcher**: Call `go app.watchTemplates(ctx)` in `main()` after creating the `App` struct.
5. **Update `handlePing`**: Change response from `{"id":%q}` with bare `startupID` to composite `startupID + "." + strconv.FormatInt(a.templateVersion.Load(), 10)`. Move `handlePing` from a package-level function to an `(a *App)` method so it can access `templateVersion`.
6. **Update `handleRootOrAsset`**: Replace inline `gettingStartedPageHTML` const with `template.ParseFiles(filepath.Join(a.templatesDir, "getting-started.html"))` + `tmpl.Execute(w, nil)`.
7. **Update `handleHost`**: Replace inline `hostPageHTML` const write with `template.ParseFiles(filepath.Join(a.templatesDir, "host.html"))` + `tmpl.Execute(w, nil)`.
8. **Update `handleLeaderboardPage`**: Replace `template.New("leaderboard").Parse(leaderboardPageTemplate)` with `template.ParseFiles(filepath.Join(a.templatesDir, "leaderboard.html"))` + same data struct.
9. **Remove** `gettingStartedPageHTML`, `hostPageHTML`, `leaderboardPageTemplate` consts from `main.go`.
10. **Error handling**: Each handler wraps parse/execute errors in `http.Error(..., 500)` + `log.Printf(...)` with file path (already the pattern in `handlePlayIndex` and `handleLeaderboardPage`).

### Phase C — Docker & Compose

1. **`app/Dockerfile`**: Add `COPY templates/ /templates` (alongside existing `COPY frontend/ /frontend`).
2. **`docker-compose.yml`** app service:
   - Add `- TEMPLATES_DIR=/templates` to environment.
   - Add `- ./templates:/templates` to volumes (bind mount for live editing).

### Phase D — Tests

1. Update/add unit tests for the three modified handlers, using `httptest.NewRecorder` + a real temp dir (`t.TempDir()`) containing minimal `.html` files. Confirm correct rendering and that the template token (`LeaderboardToken`, service URLs) is injected correctly.
2. Add a test for the watcher: write a file, advance its mtime, confirm `templateVersion` increments. (Use a short poll interval via a test-injectable parameter or a small test helper.)
3. Run `go test ./...` from `app/` to confirm all existing tests still pass.
